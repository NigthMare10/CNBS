1. Objetivo del manual

Este manual explica cómo operar la plataforma CNBS Dashboard en su primera versión, con un lenguaje orientado a usuarios administrativos, personal de supervisión, personal de validación y usuarios institucionales no técnicos.

El manual describe exclusivamente lo que el sistema hace hoy. Cuando una capacidad no existe o no forma parte del alcance actual, se explica como limitación o estado no disponible.

3. Descripción general de la plataforma

La plataforma tiene dos superficies principales:

  • Módulo administrativo: permite iniciar sesión, cargar workbooks, revisar corridas, publicar versiones y ejecutar rollback.
  • Portal público: muestra indicadores, rankings, versión activa y ficha institucional de forma honesta y controlada.

La información pública se sirve desde datasets publicados. El sitio no abre Excel en tiempo de consulta.

Regla principal
La primera versión pública solo usa dos fuentes operativas: primas y estado de situación financiera. Si un dato requiere otra fuente, el sistema lo mostrará como no disponible.

4. Requisitos para usarla

  • Acceso a la red y a los servicios locales o institucionales donde corra el sistema.
  • Credenciales válidas para el módulo admin, ya sea en modo local u OIDC.
  • Archivos Excel .xlsx válidos del dominio de primas, balance o referencia opcional.
  • Permisos acordes al rol del usuario para cargar, publicar, revertir o auditar.

5. Acceso al sistema

  1. Abra el módulo administrativo en http://localhost:3001/ o en la URL institucional configurada.
  2. Si el entorno usa modo local, ingrese usuario y contraseña.
  3. Si el entorno usa SSO, seleccione el ingreso con OIDC y complete la autenticación corporativa.
  4. Una vez autenticado, el sistema lo dirigirá a la pantalla de carga.

El portal público se consulta por separado en http://localhost:3000/ o en la URL pública definida para la institución.

7. Flujo paso a paso de operación

7.1 Iniciar sesión

  1. Ingrese al panel administrativo.
  2. Autentíquese por modo local u OIDC.
  3. Verifique que el menú superior muestre los módulos administrativos.

7.2 Cargar archivos

  1. Abra la pantalla Carga.
  2. Seleccione uno o varios archivos .xlsx.
  3. Haga clic en Iniciar ingesta.
  4. Espere a que el sistema clasifique, valide y deje la corrida en staging.

7.3 Revisar una corrida

  1. Después de cargar, el sistema lo redirige a Reconciliación.
  2. Revise la etiqueta operativa, el resumen de severidades y el diagnóstico de matching.
  3. Si necesita una vista más resumida de corridas, consulte Ingestas.

7.4 Interpretar validaciones

  • publishable: la corrida puede avanzar a publicación.
  • warningOnly: la corrida puede publicarse, pero con advertencias que deben entenderse.
  • blocked: la corrida no puede publicarse mientras existan issues críticos o altos.

7.5 Interpretar reconciliación

  • Revise reparaciones de texto y aliases resueltos.
  • Si aparecen aliases ambiguos o no resueltos, la corrida requiere intervención antes de publicar.
  • El JSON completo sirve como respaldo técnico para soporte y análisis detallado.

7.6 Publicar

  1. Abra la pantalla Publicar.
  2. Identifique la corrida deseada.
  3. Si la corrida está bloqueada, el botón aparecerá deshabilitado o reemplazado por el mensaje correspondiente.
  4. Si está permitida, seleccione Publicar corrida.
  5. El sistema generará una versión inmutable y actualizará la versión activa.

7.7 Verificar la versión activa

  1. Consulte Publicaciones o Historial en el panel admin.
  2. Verifique qué versión aparece marcada como activa.
  3. Abra luego el portal público en Versión para confirmar la metadata visible al usuario final.

7.8 Verificar datos públicos

  1. Abra el portal público.
  2. Revise Inicio, Rankings, Versión y una ficha institucional.
  3. Confirme que los bloques no soportados se muestren como no disponibles en lugar de valores inventados.

7.9 Revertir a una versión anterior

  1. Abra Publicaciones o Historial.
  2. Identifique la versión a la que desea volver.
  3. Seleccione Revertir a esta versión.
  4. Confirme luego en el portal público que la versión activa haya cambiado.

No se detectaron archivos válidosLa carga no contenía archivos útiles o el navegador envió placeholders vacíos.Repita la carga seleccionando los .xlsx reales.

11. Buenas prácticas de operación

  • Cargue únicamente workbooks .xlsx provenientes de la fuente institucional esperada.
  • Revise la etiqueta operativa, la publishability y la reconciliación antes de publicar.
  • No interprete un bloque ausente como cero; valide siempre la cobertura por dominio.
  • Conserve la lógica actual: el informe preliminar es referencia, no fuente pública.
  • Verifique la versión activa después de publicar y después de cualquier rollback.
  • Use Auditoría e Historial para reconstruir la secuencia operativa de una versión.

12. Preguntas frecuentes

12.1 ¿Puedo publicar solo el informe preliminar?

No. El informe preliminar no es fuente operativa y nunca se publica por sí solo.

12.2 ¿Por qué algunos gráficos muestran “Dato no disponible”?

Porque la fuente requerida no está presente o no permite derivación íntegra en esta versión. El sistema evita inventar datos.

12.3 ¿Qué pasa si publico una corrida con advertencias?

La versión se publica, pero queda registro de las advertencias. Deben revisarse según el contexto operativo.

12.4 ¿Qué hace el rollback?

Cambia la versión activa a una publicación anterior. No elimina datasets ni corridas históricas.

12.5 ¿El estado de resultados ya alimenta el portal público?

No. Puede detectarse y trazarse, pero no forma parte de la política operativa pública actual.
